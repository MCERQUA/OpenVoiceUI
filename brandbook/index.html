01 - Brand Identity

The face that listens.

OpenVoiceUI is a modular voice assistant platform. The brand is defined by the eye avatar, neon-on-dark aesthetics, and a cyberpunk-meets-clinical sensibility.

Mark

The Eyes

Two animated eyes with blue glow, expressive pupils, and mood states. The eyes ARE the logo. No wordmark needed in small contexts.

Wordmark

OpenVoiceUI

Space Grotesk 700. "Voice" rendered in primary blue (#0088ff). Always one word, never hyphenated.

Personality

Ambient Intelligence

Always listening, never intrusive. The UI is dark, the feedback is subtle, the voice is the primary interface. Technology that disappears.

06 - Motion

Ambient, not performative.

Animations serve feedback, not decoration. Eye blinks confirm attention. Glow pulses indicate state. Panels slide, never bounce.

Eye Blink

0.15s ease

Eyelid close/open. Confirms the system heard you.

Panel Slide

0.3s ease

Settings, transcript, and action panels slide in from edges.

Status Pulse

0.8s infinite

Listening indicator, PTT button. Rhythmic feedback during active state.

Glow Breathe

3s infinite

Eye glow intensity. Ambient presence when idle. Subtle, never distracting.

Rules: Never exceed 0.3s for UI transitions. Looping animations must have a purpose (state feedback). Disable all non-essential animations on Raspberry Pi via pi-overrides.css.

What Changed
Typography

Courier New replaced with Space Grotesk (display) and Inter (body). JetBrains Mono for code.

Icons

Emoji replaced with Lucide SVG icons. Consistent 18px, 1.5px stroke, inherits color.

Spacing

4px base grid formalized. Padding standardized to 10-12px for compact, 20-24px for panels.

Borders

Consistent 1px borders with blue rgba. Radius standardized to 8px (small), 12px (panels).

Glow

Box-shadow glow effects refined. Hover states add blue glow, active states use semantic color.

Colors

No palette changes. Same blue/cyan/green/yellow/red. Just consistent application via CSS vars.
